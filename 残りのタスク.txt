2RevenueCat設定ガイド: 管理画面で「何をどう設定すればいいか（Entitlements, Offeringsの概念）」を解説しますか？
 設定手順チェックリスト
プロジェクト作成:

RevenueCat Dashboard にログインし、「Create New Project」で「Sokkan Eisakubun」を作成。

アプリ登録 (Apps):

Project Settings > Apps から「iOS (App Store)」と「Android (Play Store)」を追加。

※Bundle IDは capacitor.config.ts に書いたもの（例: com.sokkan.eisakubun）と一致させる必要があります。

APIキーの取得:

API Keys タブから「Public SDK Key」をコピーし、.env ファイル（NEXT_PUBLIC_REVENUECAT_...）に貼り付けます。

商品と権限の設定（最重要）: RevenueCatでは以下の3階層で管理します。この通りに作成してください。

① Entitlements (権限): アプリ内で解放する「機能」の名前

作成ID: pro_access （Proプラン機能解放用）

作成ID: plus_access （Plusプラン機能解放用）

② Products (商品): App Store / Play Storeに申請する「課金アイテムID」

作成ID: sokkan_pro_monthly （Pro月額）

作成ID: sokkan_plus_monthly （Plus月額）

※まだStoreに申請していない場合でも、RevenueCat上で先に仮登録できます。

③ Offerings (販売パッケージ): Paywallに表示するセット

Default Offeringの中に、上記Productsを登録します。

★ここがポイント: コード内の実装（useRevenueCat.ts 等）では、Products ID（sokkan_pro_monthly）ではなく、Entitlements ID（pro_access）をチェックして機能を解放するようにしてください。これなら後で金額や商品IDが変わってもコード修正が不要になります。

3アイコン生成: capacitor/assets を使って、1枚の画像から全サイズのアイコンを一瞬で作る手順を教えましょうか？
Step 3: アプリアイコン・起動画面の生成
最後に、アプリの顔となるアイコンを作ります。Capacitorには便利な自動生成ツールがあります。

準備するもの
アイコン画像: icon.png (1024 x 1024 px, 背景透過なし)

起動画面画像: splash.png (2732 x 2732 px, 中央にロゴ配置)

※ロゴの周囲に十分な余白を持たせてください（スマホのサイズによってトリミングされるため）。

実行手順
プロジェクトルートに assets というフォルダを作成（既にある場合は resources などの名前になっているか確認）。

その中に上記の icon.png と splash.png を配置。

以下のコマンドを実行：

Bash

npm install @capacitor/assets --save-dev
npx capacitor-assets generate --iconBackgroundColor '#ffffff' --splashBackgroundColor '#ffffff'
※ #ffffff の部分は、アプリのブランドカラー（背景色）に合わせて変更してください。
